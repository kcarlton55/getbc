Commands on this sheet are examples only.  They are command line interface (cli) commands.  They will not work on your computer due to the path to your home directory being different.

py -m venv C:\Users\John\bc-venv

(py = python).  This command runs python and takes as arguments '-m venv', then 'C:\Users\John\bc-venv'.  The '-m' means 'module'.  Venv following the -m means that a module, i.e. a python script, named venv will be executed.  Venv came included when you installed python on your system.  The venv module is designed to create a virtual environment; that is, a separate, safe folder where your python scripts can be stored.  Venv takes at its argument 'C:\Users\John\bc-venv'.  'C:\Users\John\bc-venv' is the new folder that will be created.

Reference: https://docs.python.org/3/library/venv.html
C:\Users\John\bc-venv\Scripts\activate.bat

When a virtual environment is created, a subdirectory named Scripts and a file named activate.bat are automatically created.  By you excecuting activate.bat, later on any python commands that you issue will refer to the actived environment.  For example, any python command that you execute will be sought for in the activated environment.  When a virtual envirnment is active, the command prompt will change to look something like: (bc-venv) C:\Users\John>

'deactivate' will deactivate the environment.
py -m pip install --upgrade pip

(py = python).  Via '-m pip', python runs its pip command.  Pip got installed when you installed python on your machine.  Pip installs python scripts, usually from https://pypi.org/.  Bomcheckgui is stored there.  Here pip takes the arguments 'install --upgrade pip'.  What this does is tell pip to upgrade to its most recent version.  Upgrading pip is not always necessary, but it's just good housekeeping.

Reference: https://pypi.org/project/pip/
py -m pip install bomcheckgui

(py = python). This command runs python and takes as arguments '-m pip'.  The '-m' means 'module', and following the -m is the module that will be run, i.e. pip.  And in turn pip takes as its arguments 'install bomcheckgui'.

So then, bomcheckgui is installed.  But bomcheckgui has many modules, including bomcheck, and including another module called PyQt.  PyQt manages displaying gui windows, drag and drop features, etc.  Pip handles obtaining and installing all of these dependencies.

Reference: https://pip.pypa.io/en/stable/cli/pip_install/
mklink /h "C:\Users\John\Desktop\bomcheckgui.exe" "C:\Users\John\bc-venv\Scripts\bomcheckgui.exe"

Unlike other commands, this is not a python command.  It is a Microsoft command.  This command creates a link to the bomcheckgui program on your desktop.  Mklink means ‘make link’.  The syntax is mklink /h [link] [target].  The /h means that a hard link will be created instead of symbolic link.  (Creating a symbolic link requires administrative privileges.  It’s better to avoid this.) The name of the link that will be created is "C:\Users\John\Desktop\bomcheckgui.exe”.  The source for the link is "C:\Users\John\bc-venv\Scripts\bomcheckgui.exe".

Note that you would ordinarily have to make bomcheckgui’s virtual environment active before you could execute bomcheckgui.  But the link will cause bomcheckgui to execute directly from its virtual environment and so it is not necessary to activate it.  Any resources that bomcheckgui needs will be pulled from where it is executed.  Therefore it is not necessary to make bomcheckgui’s virtual environment active.

Reference: https://learn.microsoft.com/en-us/windows-server/administration/windows-commands/mklink/
py -m pip install --upgrade bomcheckgui

(py = python).  This will upgrade bomcheckgui if a later version of bomcheckgui is found at bomcheckgui’s home at https://pyip.org.  This is similar to upgrading pip as discussed above.  If you run this command and a later version is not available, you will get messages saying “already satisfied”.
del C:\Users\John\Desktop\bomcheckgui.exe

Delete the link to bomcheckgui that is on the Desktop.
rmdir C:\Users\John\bc-venv /s /q

This is a Microsoft command, not a python command.  It will remove the bomcheckgui’s virtual environment.  This is done by removing the directory bc-venv and all its contents including bomcheckgui.  Rmdir means remove directory.  The /s switch allows the directory to be removed even if there are files within the directory.  The /q switch means ‘quiet’; i.e. don’t prompt for confirmation before deleting files.

Reference: https://learn.microsoft.com/en-us/windows-server/administration/windows-commands/rmdir
